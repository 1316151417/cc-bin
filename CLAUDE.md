# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This repo provides helper scripts for switching Claude Code between different LLM providers (Zhipu, MiniMax, DeepSeek, Mimo) via their Anthropic-compatible APIs.

## Scripts

- **`ccs`** — Switches the global Claude Code config (`~/.claude/settings.json`) to a specified provider. Usage: `ccs <zp|mm|ds|mimo>`. Reads credentials from environment variables named `<PREFIX>_BASE_URL` and `<PREFIX>_API_KEY`.
- **`ccp`** — Launches `claude` with a per-invocation provider setting (does not modify global config). Usage: `ccp [--zp|--mm|--ds|--mimo] [claude options...]`. Writes a temporary settings file to `.claude/settings-<provider>.json`.

## Provider-to-Model mapping

| Key | Provider | Sonnet | Opus | Haiku |
|-----|----------|--------|------|-------|
| zp  | Zhipu    | GLM-5.1 | GLM-5.1 | GLM-4.5-Air |
| mm  | MiniMax  | MiniMax-M3 | MiniMax-M3 | MiniMax-M3 |
| ds  | DeepSeek | deepseek-v4-pro | deepseek-v4-pro | deepseek-v4-flash |
| mimo | Mimo    | mimo-v2.5-pro | mimo-v2.5-pro | mimo-v2.5-pro |

## Environment variables

Both scripts read credentials from `<PREFIX>_BASE_URL` and `<PREFIX>_API_KEY` env vars (e.g., `DEEPSEEK_BASE_URL`, `DEEPSEEK_API_KEY`). API endpoints are formed by appending `/anthropic` to the base URL.

## .claude/ directory

- `.claude/settings.local.json` — Local permissions (allowed git commands). Managed manually.
- `.claude/settings-ds.json` — Generated by `ccp --ds`. These provider-specific files are .gitignored.
